## Iteration 1: Fix ngrok dashboard connection error

**Task**: Fix "Dashboard unavailable: connect ECONNREFUSED 127.0.0.1:4040" error (PRD Task 5)

**Root cause**:
- ngrok v5 beta requires NGROK_AUTHTOKEN env var to connect
- Connection fails immediately when token not set
- Error shows raw error message without helpful guidance

**Solution**:
1. Added check for NGROK_AUTHTOKEN before connecting (use-ngrok.ts)
2. Return early with clear error if token missing
3. Updated StatusBar to show helpful message: "Set NGROK_AUTHTOKEN to enable remote monitoring"
4. Changed error border color from red to yellow (warning, not critical error)
5. Added fallback: always show local dashboard URL (http://localhost:3737) when ngrok unavailable

**Files changed**:
- src/hooks/use-ngrok.ts: Added NGROK_AUTHTOKEN check before connection
- src/components/StatusBar.tsx: Better error messaging + fallback to local URL

**Key decisions**:
- Make remote monitoring optional: app works fine with just local dashboard
- Treat missing ngrok token as warning (yellow) not error (red)
- Always show localhost fallback for convenience

**Tests**: All 86 tests pass
